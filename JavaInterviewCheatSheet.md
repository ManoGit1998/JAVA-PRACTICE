# Java Interview Cheat Sheet

## Basics
- Java is object-oriented and platform-independent.
- JVM executes Java bytecode.
- JDK is for development; JRE is for running Java applications.

## OOP
- Encapsulation: hide data using private variables and public methods.
- Inheritance: reuse code from parent class.
- Polymorphism: one reference can take many forms.
- Abstraction: hide implementation details.

## Important Keywords
- `static`: belongs to class
- `final`: cannot be changed
- `this`: current object
- `super`: parent class
- `abstract`: incomplete class or method
- `interface`: contract with abstract methods

## Collections
- List: ordered, duplicates allowed
- Set: unique elements only
- Map: key-value pairs

## Exceptions
- `try`: risky code
- `catch`: handles exception
- `finally`: cleanup code
- `throw`: explicitly throw exception
- `throws`: declare exception

## Multithreading
- `start()` creates a new thread
- `run()` contains task logic
- `synchronized` prevents race conditions
- `ExecutorService` is better than manually managing threads

## Java 8+
- Lambda: concise functional code
- Stream: functional-style operations on collections
- Optional: safe null handling

## Common Interview Answer Tip
Always explain concepts with a real-world example and mention trade-offs.
